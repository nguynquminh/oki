## 2024-05-24 - Search Input UX/A11y Refinement
**Learning:** In React UI components, developers often unintentionally apply `className` or `aria-label` directly to generic wrapper components or duplicate them incorrectly rather than binding them to the underlying semantic HTML element (e.g., `<input>`). Furthermore, clear buttons (`X`) that empty search inputs break keyboard navigation flow if they don't explicitly return focus to the input element they just cleared, forcing keyboard users to manually tab back to continue typing.
**Action:** When auditing search or input components, check that `aria-label` accurately describes the action to screen readers (e.g., "Xóa tìm kiếm"), ensure no duplicate ARIA attributes exist in the same component, and always bind a `useRef` auto-focus to any clear/reset actions.
